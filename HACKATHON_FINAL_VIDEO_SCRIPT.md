# Voyager Final Video Script (Simple English)

Use this while recording your screen.
Language: simple English for beginners.
Target duration: 3 min 30 sec to 4 min.

---

## 0) Before recording

1. Open the app on the Planning page.
2. Keep this sample query ready:
   `Delhi to Goa 5 days couple trip, balanced budget, good food and beach vibe`
3. Open Search page once before recording.
4. Record in full screen, zoom around 125%.

---

## 1) Time-based speaking script

### 00:00 - 00:12 | Intro
**Screen:** Home / Planning page  
**Say:**
"Hello judges. This is our prototype, **Voyager**.  
Voyager takes a simple travel sentence and turns it into a full trip plan with smart filters, AI insights, and a day-wise itinerary."

### 00:12 - 00:30 | Problem and solution
**Screen:** Cursor near search box  
**Say:**
"Usually, trip planning is split across many tabs: flights, hotels, food, and activities.  
Voyager solves this with one query and one unified dashboard."

### 00:30 - 00:55 | Natural language input
**Screen:** Paste query and click search  
**Say:**
"Now I enter this natural query: Delhi to Goa 5 days couple trip, balanced budget, good food and beach vibe.  
Our backend reads this sentence and extracts route, dates, duration, budget, and traveler preferences."

### 00:55 - 01:25 | Results dashboard
**Screen:** Search results page, slow scroll  
**Say:**
"Here we get one clear dashboard with commute, stay, activities, and dine options together.  
This gives users a complete view instead of checking many different apps."

### 01:25 - 01:55 | Smart filters
**Screen:** Open filters, change 2 to 3 options, apply  
**Say:**
"Now I use smart filters to personalize the trip.  
For example: budget range, trip pace, food style, and mobility preference.  
The filter theme is also fully consistent with the website design."

### 01:55 - 02:20 | Recommendations and insights
**Screen:** Recommendation cards + insights panel  
**Say:**
"Voyager does not only list data. It also ranks better options based on budget fit, convenience, and user interest.  
It also shows predictive insights like fare trend, weather trend, and practical travel suggestions."

### 02:20 - 02:45 | AI chat and itinerary
**Screen:** Open chat, ask for itinerary, generate itinerary  
**Say:**
"This AI chat understands current query, filters, and results context.  
In one click, it creates a day-wise itinerary with timing, estimated costs, and useful tips."

### 02:45 - 03:15 | Architecture in simple steps
**Screen:** Architecture slide  
**Say:**
"Our system works in 6 simple layers:  
1) User Layer: query and filters input.  
2) Control Layer: query parsing and flow control.  
3) Data Layer: travel data from multiple sources in parallel.  
4) Logic Layer: filtering, scoring, and ranking.  
5) AI Layer: chat support and itinerary generation.  
6) Output Layer: final dashboard and day-wise travel plan."

### 03:15 - 03:35 | Reliability and ending
**Screen:** Best UI section, slow pan  
**Say:**
"A key strength is reliability.  
Even if one external API is slow or unavailable, Voyager gives useful fallback output.  
So users always get a working plan. Thank you."

## 3) Backup lines (if demo issue happens)

If API is slow:
"Even when external APIs are slow, Voyager still keeps the user flow active using fallback logic."

If AI key is missing:
"Even without live LLM response, our system can still create a usable itinerary from available context."

---

## 4) Delivery tips

1. Speak slowly and clearly.
2. Pause 1 second after each major point.
3. Scroll slowly so judges can read.
4. Keep cursor movement clean and controlled.
5. End with confidence and smile.
